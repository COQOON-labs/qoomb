# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in Qoomb, **please do not open a public GitHub issue.**

Instead, report it through one of these channels:

- **Email:** [security@qoomb.app](mailto:security@qoomb.app)
- **GitHub Security Advisories:** [Report a vulnerability](https://github.com/COQOON-labs/qoomb/security/advisories/new)

### What to include

- Description of the vulnerability
- Steps to reproduce (if applicable)
- Affected version(s) or commit hash
- Potential impact assessment

### Our commitment

| Response              | Timeline                         |
| --------------------- | -------------------------------- |
| Acknowledgment        | ≤ 48 hours                       |
| Fix timeline provided | ≤ 7 days                         |
| Researcher credit     | Upon fix deployment (if desired) |

### Responsible disclosure

- Do **not** share vulnerabilities publicly before a fix is deployed
- Do **not** access or modify other users' data during testing
- Do **not** perform denial-of-service attacks

## Security Architecture
